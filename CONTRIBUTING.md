# Contributing to EWC Satellites Swarms Simulations

Thank you for your interest in contributing to the **Engineering World Company (EWC)** satellite simulation framework. We value precision, scalability, and collaborative engineering.

## Standards & Quality
As a project focused on satellite swarm dynamics, we prioritize:
* **Mathematical Accuracy:** Ensure all orbital mechanics and swarm logic are verified.
* **Performance:** Simulations should be optimized for high agent counts.
* **Documentation:** New features must include comments explaining the underlying physics or logic.

## How to Contribute

### 1. Reporting Issues
Use the GitHub issue tracker to report bugs or suggest architectural improvements. Please provide:
* A clear title and description.
* Steps to reproduce (for bugs).
* The expected vs. actual behavior of the swarm.

### 2. Development Workflow
1. **Fork** the repository.
2. Create a **Feature Branch** (`git checkout -b feature/your-feature-name`).
3. Implement your changes following the existing code style.
4. **Test** your changes to ensure no regressions in swarm stability.
5. Submit a **Pull Request** to the `main` branch.

### 3. Commit Messages
We prefer descriptive commit messages, e.g., `feat: add collision avoidance logic` or `fix: correct orbital decay calculation`.
